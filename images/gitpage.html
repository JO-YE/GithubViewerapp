Ready, Set, Launch: Creating a Static Site With GitHub Pages

Daniel Diaz, February 22, 2023

Share on

With all the current hot web development tools and frameworks, creating a website is getting increasingly complicated. But sometimes, you don‚Äôt need a lot of interactivity on your site. If you‚Äôre focused on getting the info to the viewer and don‚Äôt need complex functionality, a static site might be the right choice.

In this tutorial, you‚Äôll learn what a static site is, including its perks, its limitations, and how to create and deploy a simple personal website created with HTML and Bootstrap for free using GitHub Pages.

What Are GitHub Pages?

GitHub is a web-based platform for hosting Git repositories and collaborating on software projects. It offers tools for sharing and tracking code changes, managing and reviewing code, and the ability to open and review pull requests.

Do not confuse Git and GitHub! GitHub is a hosting service that allows collaboration between developers, whereas Git is the local version control software you use to save snapshots of the state of your software project.

GitHub Pages is one of the best features of GitHub. It is a service that allows you to host a static website directly from a GitHub repository. This means you can use your repository to store your website‚Äôs code and files, and GitHub will automatically publish them as a website you can access online.

In summary, GitHub Pages is a quick and easy way to get your website up and running, and it‚Äôs especially useful for showcasing your portfolio, open-source projects, or other static content.

Static vs Dynamic Websites

As we‚Äôve seen, GitHub Pages provides a way to deploy static websites. But what is the difference between a static website and a dynamic website?

Let‚Äôs start by discussing the content on said sites.

Static content refers to the elements of the website that remain the same for all users, regardless of who they are or what actions they take on the site. This can include things like the text, images, and layout of the website, as well as the underlying code and files that make up the site. A static site is delivered to the user exactly as it‚Äôs stored.

In contrast, dynamic content is content that changes based on the user‚Äôs actions ‚Äî like signing in, interacting with a paywall, or commenting on a post ‚Äî or other factors, such as the current time or location.

For example, a website that displays a still image of a product will always show the same image to every user (static). On the other hand, a website that displays the current time will show a different time to each user based on their location (dynamic).

Generally, we can say that a website is static if only contains HTML, CSS, and JavaScript on the frontend, with no server technologies like PHP or Python that interact with a database.

While it‚Äôs not possible to build dynamic websites using GitHub Pages, you can use easily build your own using a CMS such as WordPress, or static site generators like Gatsby or Hugo.

GitHub Pages Key Features

Let‚Äôs see the strengths of GitHub Pages as a hosting service:

  1. Easy setup and publishing: GitHub Pages makes it easy to get started with just a few simple steps. You can enable GitHub Pages for your repository and specify the source for your website files, and GitHub will automatically publish your website and make it available at a URL based on your username and repository name.
  2. Custom domains: With GitHub Pages, you can use a custom domain name for your website instead of the default URL provided by GitHub. This allows you to use your own branding and make it easier for users to find and access your website.
  3. HTTPS support: GitHub Pages offers support for HTTPS, which allows for secure connections to your website. This is crucial for building the trust of your site.
  4. Jekyll support: GitHub Pages supports Jekyll, a static site generator that allows you to create sophisticated websites using templates and other features. This makes it easy to create professional-looking websites without having to write all of the code from scratch.

Limitations

As stated before you can only create static sites with GitHub Pages. If you want to build a complex project with many functionalities you would need other hosting services. You should also bare in mind, that you can‚Äôt use GitHub Pages for commercial purposes, such as running an online business, or ecommerce.

GitHub Pages doesn‚Äôt allow your site to be larger than 1 GB, meaning the files of your repository can‚Äôt surpass that amount of memory. Most of the time, 1 GB is more than enough for a static site; just be sure to compress your images.

It also has a soft bandwidth limit of 100 GB per month. This amount of bandwidth would be enough to distribute your website to a few thousand people per month, so unless you have a huge audience, you‚Äôll be good to go.

Creating and Deploying With GitHub Pages: Step-by-Step Guide

Creating a GitHub Page is a simple and straightforward process to host a static site. Bare in mind that if you need some kind of database connection, you should have an external database provider.

In the following guide, you‚Äôll learn how to create a GitHub Page from scratch. That includes creating a remote repository, building a responsive personal website with HTML, and deploying to the web with GitHub.

Let‚Äôs get into it!

1. Sign Up on GitHub

To kick things off, you must have an active GitHub account. If you don‚Äôt have one, go to their signup page. It shouldn‚Äôt take more than a couple of minutes to fill in the form.

After signing in, you should be able to create a remote repository.

2. Create a Remote Repository

A repository is a directory where you store all the code related to a particular project.

From the GitHub home page, click on the ‚ÄúNew‚Äù or ‚ÄúCreate repository‚Äù button located in the left panel of the site. This will redirect you to a form where you‚Äôll fill in your repo‚Äôs information.

These next steps are crucial:

  1. Set the name of your repository to "yourusername".github.io.
  2. Check the public button. You must set the repository to Public to publish your site.
  3. Add a README.

You can only have a repository for a given personal account or organization. That‚Äôs why the repository‚Äôs name is your username and the github.io domain. Later, we‚Äôll see how to set up a site from a repository.

Unless you have GitHub Pro, you can only publish a GitHub page if the repository is public. Keep this in mind if you don‚Äôt want to share your site‚Äôs source code publicly.

After this, you should have something like the following:

If you already have the working source code of your site, you can skip the common Git workflow and drop the files right into the repo instead.

To do this, click the Add file menu in your repository, and select the option Upload files. Then select the files of your website, with the main HTML file named index.html. Remember to put all your CSS and JavaScript files in the repository as well.

Finally, hit the Commit changes button.

In the following section, we‚Äôll be building a simple personal website with HTML and Bootstrap. Then we‚Äôll upload it to GitHub and set it up as a GitHub public page with a custom domain.

3. Build a Personal Site

We‚Äôll start off by cloning the GitHub repo we just created. To do this, make sure you have the Git client already installed on your computer. (If you don‚Äôt, take a look at our tutorial on Git and GitHub.)

On your terminal, type git clone and the URL you copied. This will download and create a copy of the remote repository on your local machine so you can build your website.

Then enter the new folder named yourusername.github.io with cd and ls. You should see the .git folder, which contains the configuration and metadata of your project, as well as the README.md file if you created one.

4. Publishing a User GitHub Page

As soon as you push the index.html to the remote repository named after your username, GitHub will automatically start a workflow process to set up your site online. It may take a couple of minutes, but you‚Äôll get your static site up and running automatically.

The URL of your site will be something like the following: https://kinstauser.github.io/

If after 10 minutes your site is not online, you can try to make a dummy change to your code (e.g., adding a space) and push again to reactivate the GitHub Pages‚Äô building process.

Important

You can‚Äôt unpublish a user-level GitHub page. If you want to do this, you‚Äôll have to change the repo‚Äôs name.

5. Publishing a Repository GitHub Page

Until now, we‚Äôve created a user site, but what if we want to have multiple published GitHub sites? Then we must go with a project site.

As an example, we‚Äôll use the HTML technology site we created in the Git for Web development tutorial.

The easiest way is to go to the Settings tab of our repository, then to the Pages option inside the ‚ÄúCode and automation‚Äù section.

Select the source Deploy from a branch, and click on the branch you want to deploy the files from. It‚Äôs vastly recommended to publish from the main branch, but create features and fix bugs using auxiliary branches, and then merge them. This way you won‚Äôt introduce errors to the published site so easily.

Once you‚Äôve selected the branch, select the folder you want to serve the files from ‚Äî usually the root (/) ‚Äî and click save.

Again, wait for a few minutes. Your site should be available at "yourusername".github.io/.

To unpublish a repository site, you can go to Settings, then Pages, and click on the three dots option. You‚Äôll see a box with the message Unpublish site.

Great! You have the site of your open-source project up and running, but the domain name itself is lengthy and certainly not easy to remember. Let‚Äôs see how we can improve this.

6. Setting Up a Custom Domain

The easiest way to set up a custom domain for a GitHub page and ensure that it works is to go to your DNS provider and create four A records for the IP addresses of GitHub Pages:

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

You must also set up a CNAME record with yourusername.github.io as the target. Usually, DNS changes are slow, so be patient it, could take up to a whole day.

Info

If you need more information on how to create these records, please refer to your DNS provider‚Äôs guides.

After doing this, go to the Custom domain section in the settings of your repo, type in your domain, click on the Save button, and wait for GitHub to check your custom domain.

Also, if your DNS supports it, check the Enforce HTTPS box to only serve your site over HTTPS.

Congrats! If you‚Äôve followed along up to this point of the tutorial, you have a static website hosted in GitHub Pages for free.

Best Practices for GitHub Pages

Before we part ways, here are some best practices you should take into account when publishing a GitHub site:

  1. Never make direct commits to the branch you‚Äôre deploying from. Create changes in other branches, then create a pull request.
  2. Select a good domain for your site if you can afford it. It‚Äôs one of the most important decisions for your personal or project brand.
  3. Don‚Äôt use GitHub Pages for commercial purposes, such as spinning up an ecommerce site.
  4. Evaluate your needs. It‚Äôs great to be able to publish a static site for free, but if you‚Äôre expecting a lot of traffic or want complex features, paying for a great hosting service is the way to go.
